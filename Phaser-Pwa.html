# Phaser + Pwa 创建离线H5游戏

PWA全称Progressive Web App，即渐进式WEB应用。

主要功能：

  • 可以添加至主屏幕，点击主屏幕图标可以实现启动动画以及隐藏地址栏
  • 实现离线缓存功能，即使用户手机没有网络，依然可以使用一些离线功能
  • 实现了消息推送

# 项目准备

为了快速体验效果，项目使用我之前使用Phaser开发的一个套圈圈游戏，因为并不需要更改原游戏代码，所以使用打包后的版本，目录只有css,js,img和一个index.html文件。

# 添加 service worker

service worker可以理解为一个代理，在浏览器后台运行，可以用于拦截网站的HTTP请求，缓存和检索缓存资源，并且可以提供推送通知，所以我们可以使用它来缓存游戏资源，使H5能够离线访问。

service workder 必须运行在HTTPS服务器上，本地可以使用localhost调试

现在开始给我们的游戏添加service worker

在根目录添加一个sw.js的文件,先让文件空着，下一步打开index.html在页面引入JS的上方添加以下代码

<script>
    if ('serviceWorker' in navigator) {
        window.addEventListener('load', function() {
            navigator.serviceWorker.register('/sw.js', {scope: '/'}).then(function(registration) {
                console.log('ServiceWorker registration successful with scope: ', registration.scope);
            }, function(err) {
                console.log('ServiceWorker registration failed: ', err);
            });
        });
    }
</script>

我们可以通过Chrome开发人员工具中的"Application"选项卡来验证service worker是否已安装

# 缓存资源

确保service worker注册成功后吗,在刚才的sw.js文件中添加以下代码


var cacheName = 'phaser-pwa';
var filesToCache = [
    '/',
    '/js/game.js',
    '/css/game.css',
    '/index.html',
    '/img/logo.png',
    //此处省略了其它资源
];
 
self.addEventListener('install', function(event) {
  console.log('sw install');
  event.waitUntil(
    caches.open(cacheName).then(function(cache) {
      console.log('sw caching files');
      return cache.addAll(filesToCache);
    }).catch(function(err) {
      console.log(err);
    })
  );
});

self.addEventListener('fetch', (event) => {
  console.log('sw fetch');
  console.log(event.request.url);
  event.respondWith(
    caches.match(event.request).then(function(response) {
      return response || fetch(event.request);
    }).catch(function (error) {
      console.log(error);
    })
  );
});

先看下以上代码做了些什么

  • 声明两个变量cacheName 和 filesToCache 用于储存缓存名称和游戏使用的资源
  • 监听install事件，它发生在浏览器安装并注册 Service Worker 时触发，在回调函数中调用waitUntil函数
  • event.waitUntil（）方法接收一个Promise参数，根据Promise确认是否安装成功
  • caches.open() 用于打开浏览器的的缓存，cache.addAll保存我们的资源到浏览器，所有文件缓存成功后service worker的状态将会改为installed
  • 为fetch事件添加一个监听，接下来，使用 caches.match() 函数来检查传入的请求 URL 是否匹配当前缓存中存在的任何内容。如果存在的话，返回缓存的资源。如果资源并不存在于缓存当中，通过网络来获取资源，并将获取到的资源添加到缓存中。

# 添加到主屏幕

让浏览器显示添加到主屏幕提示，我们需要监听beforeinstallprompt事件，在chrome68之前浏览器会自动弹出添加主屏幕的提示框，之后chrome取消了这个功能，所以我们必须自己写一个弹框提示。

为了让chrom启动beforeinstallprompt，需要满足以下几个条件

  • 这个app没有被安装过
  • 网站需要包含一个manifest文件，并包含以下内容
    • short_name 或者 name
    • 必须包含192px和512px大小的图标
    • start_url
    • display：fullscreen, standalone, minimal-ui 3个中其中一个
  • HTTPS服务器
  • 已成功注册service worker

# 添加manifest.js

先将manifest.js添加到项目跟目录

{
    "name": "taoquan",
    "display": "fullscreen",
    "orientation": "portrait",
    "Scope": "/",
    "start_url": "/",
    "icons": [
        {
            "src": "icon/icon-192x192.png",
            "sizes": "192x192",
            "type": "image/png"
        }
    ]
}

TIP

icons是必填，大小在192到512之间，可以用ps随便保存一张图片

清单创建完后打开index.html 在<head>标签中引入manifest.json

<link rel="manifest" href="manifest.json" />

在Application中选择Manifest确认是否引入成功

在body顶部添加一个弹框

    <style>
        .addScreenModal {
            display: none;
            position: absolute;
            width: 60%;
            height: 100px;
            z-index: 1000;
            left: 50%;
            top: 50%;
            transform: translateX(-50%)translateY(-50%);
            background: #fff;
        }
        ...
    </style>
    <div id="myModal" class="addScreenModal">
        <!-- Modal content -->
        <span class="close">&times;</span>
        <div class="modal-content">
            <p>添加到主屏幕？</p>
            <a onclick="offlinePrompt()">安装</a>
        </div>
    </div>

在刚才添加注册service worker代码下面继续添加

let deferredPrompt;
window.addEventListener('beforeinstallprompt', function (e) {
    console.log('beforeinstallprompt triggered');
    e.preventDefault();
    deferredPrompt = e;
    modal.style.display = 'block';
});

var modal = document.getElementById('myModal');

var span = document.getElementsByClassName('close')[0];

window.onclick = function(event) {
    if (event.target == modal) {
        modal.style.display = 'none';
    }
}

span.onclick = function() {
    modal.style.display = 'none';
}
function offlinePrompt() {
    deferredPrompt.prompt();
}
